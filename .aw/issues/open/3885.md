Parent epic: #3882 — SDD takeover, managed layer.

## Capability Alignment

Standardization → managed layer. This is the prerequisite for Sections 1–6 of #3882: the per-subdirectory marker sweeps need a batch-marker helper and a usable `aw standardize managed next mamba` driver before they can run at scale.

## Acceptance Criteria

1. `aw standardize managed next mamba` returns one uncovered file path + a default-marker suggestion (CODEGEN candidate vs. HANDWRITE candidate) based on heuristics over the file's contents (e.g. presence of `// SPEC-REF:` comment, generated-by header, hand-edit indicators).
2. A batch-marker helper script (path TBD, likely `projects/mamba/scripts/standardize/mark-batch.sh`) accepts a glob + a marker kind and writes the marker to every match in one commit-ready diff. Dry-run mode lists matches without writing.
3. Helper refuses to mark a file that already has a marker, unless `--force` is passed (with a confirmation prompt).
4. `aw standardize managed report mamba` covers at least 2 files marked by the helper in dry-run + apply mode (smoke test).

## Agent Estimate

agent_minutes=120, confidence=medium, risk=medium, human_attention=none.

## Out of scope

Actual marker sweeps for any subdirectory — those are Sections 1–6.
