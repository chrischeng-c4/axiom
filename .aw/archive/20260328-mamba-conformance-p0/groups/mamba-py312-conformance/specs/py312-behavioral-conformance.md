# Py312 Behavioral Conformance

## Overview

Comprehensive Py3.12 behavioral conformance test suite covering builtins, stdlib, and language features NOT covered by sibling specs (generator-iterator-conformance, data-structure-conformance). Covers #1037.

| Domain | Fixture Dir | Operations Covered | Existing Fixtures |
|--------|-------------|-------------------|-------------------|
| Builtins (108 functions) | `builtins/` | abs, all, any, bin, bool, chr, dict, dir, enumerate, filter, float, format, getattr, hasattr, hash, hex, id, int, isinstance, issubclass, iter, len, list, map, max, min, next, oct, open, ord, pow, print, range, repr, reversed, round, set, setattr, sorted, str, sum, tuple, type, zip | 10 (numeric, type_builtins, type_conversions, collection_builtins, functional, io_builtins, iteration, object_protocol, sequence, string_builtins) |
| Stdlib top-16 | `stdlib/` | json, os, re, datetime, collections, pathlib, math, sys, io, csv, hashlib, itertools, functools, struct, random, asyncio | 16 dirs (partial coverage) |
| Language: decorators | `language/` | stacked decorators, parameterized, class decorators, functools.wraps, decorator return value | 3 (decorators, decorator_return, decorator_with_args/) |
| Language: class system | `class_system/` | inheritance, super(), descriptors, MRO, magic methods | 3 (inheritance, super_call, descriptors) |
| Language: exceptions | `exceptions/` | hierarchy, chaining, custom, matching, ExceptionGroup, except* | 5 (hierarchy, chaining, custom, matching, exception_group) |
| Language: pattern matching | `language/` | match/case, structural patterns, guards, OR patterns | 1 (pattern_matching) |
| Language: comprehensions | `language/` | list/dict/set comprehension scope (PEP 709), nested | 1 (comprehension_scope) |
| Language: f-strings | `language/` | nested f-strings (PEP 701), advanced f-string features | 3 (fstring_advanced, fstring_nested, nested_fstring) |
| Language: context managers | `language/` | with statement, multiple context managers, exception handling | 1 (context_managers) |
| Language: lambda | `language/` | lambda expressions, closures, nested lambdas | 1 (lambda_expressions) |
| Test framework | CLI | `cclab mamba test --conformance` runner, golden file comparison, xfail directives | 1 (conformance_tests.rs) |

**Test approach**: Each `.py` fixture runs through the full Mamba JIT pipeline (parse ‚Üí typecheck ‚Üí HIR ‚Üí MIR ‚Üí Cranelift JIT ‚Üí capture stdout). Output is compared against `.expected` golden files generated by CPython 3.12. Any diff = conformance bug.

**Scope**: This spec adds edge-case fixtures for builtins (error branches, type coercion, boundary values), stdlib modules (complete public API coverage for top-16 modules), language features (decorator edge cases, class MRO conflicts, exception chaining, advanced pattern matching), and the automated CLI runner `cclab mamba test --conformance`. Generator/iterator and data structure conformance are covered by sibling specs.

### Constraints

- All existing conformance fixtures must continue to pass (no regressions)
- Test harness: `.py` fixture + `.expected` golden file, run via `conformance_tests.rs`
- Golden files generated by running fixtures under CPython 3.12
- Each new fixture must cover edge cases not in existing fixtures
- No changes to runtime implementation in this spec ‚Äî test-only; implementation bugs found are tracked as separate issues
- Generator/iterator conformance covered by sibling spec `generator-iterator-conformance`
- Data structure conformance covered by sibling spec `data-structure-conformance`
- Async generators/asyncio event loop remain unsupported (xfail ‚Äî #800, #801)
## Scenarios

### S1: Numeric builtins ‚Äî edge cases (R1)

**GIVEN** `print(abs(-0.0)); print(abs(float('inf'))); print(round(2.5)); print(round(3.5)); print(divmod(7, 3)); print(pow(2, 10)); print(pow(2, -1))`
**WHEN** executed through Mamba JIT
**THEN** `0.0`, `inf`, `2`, `4`, `(2, 1)`, `1024`, `0.5` ‚Äî matches CPython banker's rounding and edge cases

### S2: Numeric builtins ‚Äî bin/hex/oct formatting (R1)

**GIVEN** `print(bin(255)); print(hex(255)); print(oct(255)); print(bin(-1)); print(hex(-1))`
**WHEN** executed
**THEN** `0b11111111`, `0xff`, `0o377`, `-0b1`, `-0x1`

### S3: Type introspection ‚Äî isinstance/issubclass chains (R2)

**GIVEN** `print(isinstance(True, int)); print(isinstance(True, (str, int))); print(issubclass(bool, int)); print(issubclass(int, object))`
**WHEN** executed
**THEN** `True`, `True`, `True`, `True` ‚Äî bool is subclass of int in CPython

### S4: Type introspection ‚Äî getattr/setattr/delattr (R2)

**GIVEN** `class C: x = 10`; `obj = C(); print(getattr(obj, 'x')); setattr(obj, 'y', 20); print(obj.y); delattr(obj, 'y); print(hasattr(obj, 'y'))`
**WHEN** executed
**THEN** `10`, `20`, `False`

### S5: Type introspection ‚Äî getattr with default (R2)

**GIVEN** `print(getattr(object(), 'missing', 'default_val'))`
**WHEN** executed
**THEN** `default_val` ‚Äî getattr with default doesn't raise AttributeError

### S6: String/repr builtins ‚Äî repr for all types (R3)

**GIVEN** `print(repr(42)); print(repr('hello')); print(repr([1,2])); print(repr(None)); print(repr(True))`
**WHEN** executed
**THEN** `42`, `'hello'`, `[1, 2]`, `None`, `True`

### S7: String/repr builtins ‚Äî chr/ord round-trip (R3)

**GIVEN** `print(chr(65)); print(chr(8364)); print(ord('A')); print(ord('\u20ac'))`
**WHEN** executed
**THEN** `A`, `‚Ç¨`, `65`, `8364`

### S8: Collection builtins ‚Äî sorted with key (R4)

**GIVEN** `print(sorted([3,1,4,1,5])); print(sorted('hello')); print(sorted([3,1,2], reverse=True)); print(sorted(['banana','apple'], key=len))`
**WHEN** executed
**THEN** `[1, 1, 3, 4, 5]`, `['e', 'h', 'l', 'l', 'o']`, `[3, 2, 1]`, `['apple', 'banana']`

### S9: Collection builtins ‚Äî all/any with generators (R4)

**GIVEN** `print(all([True, True, True])); print(all([True, False, True])); print(any([False, False, True])); print(any([])); print(all([]))`
**WHEN** executed
**THEN** `True`, `False`, `True`, `False`, `True` ‚Äî empty iterable: all=True, any=False

### S10: Collection builtins ‚Äî min/max with key and default (R4)

**GIVEN** `print(min([3,1,4], key=lambda x: -x)); print(max([], default='empty')); print(min('hello'))`
**WHEN** executed
**THEN** `4`, `empty`, `e`

### S11: Print with sep/end kwargs (R5)

**GIVEN** `print(1, 2, 3, sep='-'); print('hello', end='!!!\n'); print('a', 'b', sep='', end='')`
**WHEN** executed
**THEN** `1-2-3`, `hello!!!`, `ab` (no trailing newline for last)

### S12: math module ‚Äî basic functions (R6)

**GIVEN** `import math; print(math.floor(3.7)); print(math.ceil(3.2)); print(math.sqrt(16)); print(math.factorial(5)); print(math.gcd(12, 8))`
**WHEN** executed
**THEN** `3`, `4`, `4.0`, `120`, `4`

### S13: math module ‚Äî special values (R6)

**GIVEN** `import math; print(math.isnan(math.nan)); print(math.isinf(math.inf)); print(math.isfinite(42.0)); print(math.pi); print(math.e)`
**WHEN** executed
**THEN** `True`, `True`, `True`, `3.141592653589793`, `2.718281828459045`

### S14: json module ‚Äî dumps/loads round-trip (R7)

**GIVEN** `import json; d = {'name': 'test', 'values': [1, 2, 3]}; s = json.dumps(d, sort_keys=True); print(s); print(json.loads(s) == d)`
**WHEN** executed
**THEN** `{"name": "test", "values": [1, 2, 3]}`, `True`

### S15: json module ‚Äî indent and special values (R7)

**GIVEN** `import json; print(json.dumps({'a': 1}, indent=2)); print(json.dumps(float('inf')))`
**WHEN** executed
**THEN** formatted JSON with 2-space indent; `Infinity` (or raises ValueError depending on allow_nan)

### S16: re module ‚Äî match/search/findall (R8)

**GIVEN** `import re; print(re.match(r'\d+', '123abc').group()); print(re.search(r'[a-z]+', '123abc').group()); print(re.findall(r'\d+', 'a1b2c3'))`
**WHEN** executed
**THEN** `123`, `abc`, `['1', '2', '3']`

### S17: re module ‚Äî sub and split (R8)

**GIVEN** `import re; print(re.sub(r'\d', 'X', 'a1b2c3')); print(re.split(r'[,;]', 'a,b;c,d'))`
**WHEN** executed
**THEN** `aXbXcX`, `['a', 'b', 'c', 'd']`

### S18: collections ‚Äî Counter and defaultdict (R9)

**GIVEN** `from collections import Counter, defaultdict; c = Counter('abracadabra'); print(c.most_common(3)); dd = defaultdict(int); dd['x'] += 1; print(dd['x']); print(dd['y'])`
**WHEN** executed
**THEN** `[('a', 5), ('b', 2), ('r', 2)]`, `1`, `0`

### S19: collections ‚Äî deque and namedtuple (R9)

**GIVEN** `from collections import deque, namedtuple; d = deque([1,2,3], maxlen=3); d.appendleft(0); print(list(d)); Point = namedtuple('Point', ['x', 'y']); p = Point(1, 2); print(p.x, p.y)`
**WHEN** executed
**THEN** `[0, 1, 2]`, `1 2` ‚Äî deque drops rightmost when maxlen exceeded

### S20: datetime ‚Äî construction and arithmetic (R10)

**GIVEN** `from datetime import datetime, timedelta; d = datetime(2024, 1, 15, 10, 30); print(d.strftime('%Y-%m-%d')); d2 = d + timedelta(days=10); print(d2.day)`
**WHEN** executed
**THEN** `2024-01-15`, `25`

### S21: itertools ‚Äî chain, islice, product (R12)

**GIVEN** `from itertools import chain, islice, product; print(list(chain([1,2], [3,4]))); print(list(islice(range(10), 2, 7, 2))); print(list(product('ab', '12')))`
**WHEN** executed
**THEN** `[1, 2, 3, 4]`, `[2, 4, 6]`, `[('a', '1'), ('a', '2'), ('b', '1'), ('b', '2')]`

### S22: functools ‚Äî reduce, partial, lru_cache (R12)

**GIVEN** `from functools import reduce, partial; print(reduce(lambda a,b: a+b, [1,2,3,4])); add5 = partial(lambda a,b: a+b, 5); print(add5(3))`
**WHEN** executed
**THEN** `10`, `8`

### S23: Decorator ‚Äî stacked, bottom-up application (R14)

**GIVEN** `def d1(f): print('d1'); return f`; `def d2(f): print('d2'); return f`; `@d1 @d2 def foo(): pass`
**WHEN** executed
**THEN** `d2` printed first, then `d1` ‚Äî bottom-up application order

### S24: Decorator ‚Äî parameterized decorator (R14)

**GIVEN** `def repeat(n): def decorator(f): def wrapper(*a): for _ in range(n): f(*a); return wrapper; return decorator`; `@repeat(3) def greet(): print('hi')`; `greet()`
**WHEN** executed
**THEN** `hi` printed 3 times

### S25: Decorator ‚Äî functools.wraps preserves metadata (R14)

**GIVEN** `from functools import wraps; def my_deco(f): @wraps(f) def wrapper(*a, **kw): return f(*a, **kw); return wrapper`; `@my_deco def my_func(): """docstring"""; pass`; `print(my_func.__name__)`
**WHEN** executed
**THEN** `my_func` ‚Äî not `wrapper`

### S26: Class system ‚Äî C3 MRO with diamond (R15)

**GIVEN** `class A: pass; class B(A): pass; class C(A): pass; class D(B, C): pass; print([cls.__name__ for cls in D.__mro__])`
**WHEN** executed
**THEN** `['D', 'B', 'C', 'A', 'object']` ‚Äî C3 linearization

### S27: Class system ‚Äî staticmethod and classmethod (R15)

**GIVEN** `class Foo: @staticmethod def s(): return 'static'; @classmethod def c(cls): return cls.__name__`; `print(Foo.s()); print(Foo.c())`
**WHEN** executed
**THEN** `static`, `Foo`

### S28: Exception chaining ‚Äî raise from (R16)

**GIVEN** `try: try: 1/0 except ZeroDivisionError as e: raise ValueError('bad') from e except ValueError as e: print(type(e.__cause__).__name__)`
**WHEN** executed
**THEN** `ZeroDivisionError` ‚Äî `__cause__` preserves original exception

### S29: Pattern matching ‚Äî structural patterns (R17)

**GIVEN** `match {'action': 'move', 'x': 10, 'y': 20}: case {'action': 'move', 'x': x, 'y': y}: print(f'move to {x},{y}'); case _: print('unknown')`
**WHEN** executed
**THEN** `move to 10,20` ‚Äî mapping pattern with capture

### S30: Pattern matching ‚Äî OR and guard (R17)

**GIVEN** `match 42: case 1 | 2 | 3: print('small'); case x if x > 40: print(f'big: {x}'); case _: print('other')`
**WHEN** executed
**THEN** `big: 42` ‚Äî OR pattern not matched, guard pattern matches

### S31: Comprehension scope ‚Äî PEP 709 (R18)

**GIVEN** `x = 'outer'; result = [x for x in range(3)]; print(x); print(result)`
**WHEN** executed
**THEN** `outer`, `[0, 1, 2]` ‚Äî comprehension variable does NOT leak to enclosing scope

### S32: Context manager ‚Äî exception suppression (R20)

**GIVEN** `class Suppress: def __enter__(self): return self; def __exit__(self, *args): return True`; `with Suppress(): raise ValueError('suppressed')`; `print('after')`
**WHEN** executed
**THEN** `after` ‚Äî `__exit__` returning True suppresses the exception

### S33: Context manager ‚Äî multiple managers (R20)

**GIVEN** `class CM: def __init__(self, n): self.n = n; def __enter__(self): print(f'enter {self.n}'); return self; def __exit__(self, *a): print(f'exit {self.n}'); return False`; `with CM(1), CM(2): print('body')`
**WHEN** executed
**THEN** `enter 1`, `enter 2`, `body`, `exit 2`, `exit 1` ‚Äî LIFO exit order

### S34: Lambda ‚Äî closure over loop variable (R21)

**GIVEN** `fns = [lambda x=i: x for i in range(3)]; print([f() for f in fns])`
**WHEN** executed
**THEN** `[0, 1, 2]` ‚Äî default arg captures loop variable at each iteration

### S35: Conformance CLI runner (R22)

**GIVEN** all conformance fixtures in `tests/fixtures/conformance/`
**WHEN** `cclab mamba test --conformance` is executed
**THEN** runner discovers all `.py` files, executes each, compares against `.expected`, prints pass/fail/xfail summary, exits 0 on all-pass
