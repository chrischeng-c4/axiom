# Data Structure Conformance

## Overview

Conformance test suite ensuring all Mamba data structure operations (list, dict, set, tuple, str, bytes) produce identical results to CPython 3.12. Covers #759.

| Type | Impl Module | Methods Covered | Fixture Count |
|------|-------------|-----------------|---------------|
| list | `runtime/list_ops.rs` | 22 (append, extend, insert, pop, remove, sort, reverse, clear, copy, index, count, contains, len, concat, repeat, eq, getitem, setitem, delitem, slice, slice_full, from_iterable) | 3 (list_methods, list_slicing, list_compare) |
| dict | `runtime/dict_ops.rs` | 18 (getitem, get, setitem, delitem, contains, len, keys, values, items, pop, setdefault, update, clear, copy, popitem, eq, merge, from_pairs) | 2 (dict_methods, dict_comprehension) |
| set | `runtime/set_ops.rs` | 15 (add, remove, discard, contains, len, union, intersection, difference, symmetric_difference, clear, copy, issubset, issuperset, from_list, new) | 1 (set_ops) |
| tuple | `runtime/tuple_ops.rs` | 16 (getitem, slice, slice_full, len, contains, count, index, concat, repeat, eq, hash, lt, le, gt, ge, from_iterable) | 1 (tuple_ops) |
| str | `runtime/string_ops.rs` | 38 (upper, lower, capitalize, title, swapcase, strip, lstrip, rstrip, find, rfind, count, startswith, endswith, contains, replace, split, join, isdigit, isalpha, isalnum, isspace, isupper, islower, istitle, center, ljust, rjust, zfill, encode, hash, eq, lt, splitlines, partition, rpartition, removeprefix, removesuffix, format) | 1 (string_methods) |
| bytes | `runtime/bytes_ops.rs` | 18 (new, getitem, len, decode, hex, find, concat, contains, count, startswith, endswith, replace, slice_full, split, join, strip, lstrip, rstrip) | 1 (bytes_ops) |

**Test approach**: Each `.py` fixture in `tests/fixtures/conformance/data_structures/` is executed through the full Mamba JIT pipeline (parse → typecheck → HIR → MIR → Cranelift JIT → capture stdout). Output is compared against `.expected` golden files generated by CPython 3.12. Any diff = conformance bug.

**Scope**: Existing fixtures cover core operations. This spec adds comprehensive edge-case fixtures to reach full CPython 3.12 parity for all methods in each runtime module.
## Requirements

| ID | Type | Title | Priority | Acceptance Criteria |
|----|------|-------|----------|---------------------|
| R1 | list | List mutation methods match CPython | P0 | `append`, `extend`, `insert`, `pop`, `pop(i)`, `remove`, `sort`, `sort(key=, reverse=)`, `reverse`, `clear`, `copy` produce identical stdout to CPython 3.12 |
| R2 | list | List query methods match CPython | P0 | `index`, `count`, `len`, `in`/`not in`, `==`/`!=`/`<`/`<=`/`>`/`>=` produce identical results |
| R3 | list | List slicing and indexing match CPython | P0 | `a[i]`, `a[-i]`, `a[start:stop]`, `a[::step]`, `a[::-1]`, `a[start:stop:step]` produce identical results; negative indices, out-of-range slices handled identically |

| R7 | set | Set mutation methods match CPython | P0 | `add`, `remove`, `discard`, `clear`, `copy` — identical results; `remove` raises `KeyError` on missing element |
| R8 | set | Set algebra operations match CPython | P0 | `\|`, `&`, `-`, `^` operators and method forms (`union`, `intersection`, `difference`, `symmetric_difference`), `issubset`, `issuperset`, set comprehension, `frozenset` — identical results |
| R9 | tuple | Tuple operations match CPython | P0 | indexing, negative indexing, slicing, `len`, `in`, `count`, `index`, `+` concat, `*` repeat, `==`/`<`/`<=`/`>`/`>=` comparison, `hash`, single-element `(x,)`, empty `()` — identical results |

| R11 | str | String indexing and slicing match CPython | P0 | `s[i]`, `s[-i]`, `s[start:stop]`, `s[::step]`, `s[::-1]` — identical results |

### Constraints

- All existing conformance fixtures in `tests/fixtures/conformance/data_structures/` must continue to pass
- Test harness: `.py` fixture + `.expected` golden file, run via `conformance_tests.rs`
- Golden files generated by running fixtures under CPython 3.12
- Each new fixture must cover edge cases not in existing fixtures (empty containers, boundary indices, error conditions)
- No changes to runtime implementation in this spec — test-only; implementation bugs found are tracked as separate issues
## Scenarios

### S1: List — mutation methods (R1)

**GIVEN** `a = [1, 2, 3]; a.append(4); a.extend([5, 6]); a.insert(0, 0); x = a.pop(); a.remove(3); a.reverse(); b = a.copy(); a.clear()`
**WHEN** executed through Mamba JIT
**THEN** each `print()` output matches CPython 3.12 exactly (element order, repr format)

### S2: List — sort with key and reverse (R1)

**GIVEN** `a = [3, 1, 4, 1, 5]; a.sort(); print(a); a.sort(reverse=True); print(a); b = ['banana', 'apple', 'cherry']; b.sort(key=len); print(b)`
**WHEN** executed
**THEN** `[1, 1, 3, 4, 5]`, `[5, 4, 3, 1, 1]`, `['apple', 'banana', 'cherry']`

### S3: List — index/count edge cases (R2)

**GIVEN** `a = [1, 2, 2, 3]; print(a.index(2)); print(a.count(2)); print(a.count(99))`
**WHEN** executed
**THEN** `1`, `2`, `0` — `index` returns first occurrence, `count` returns 0 for absent

### S4: List — negative index and out-of-range slice (R3)

**GIVEN** `a = [10, 20, 30]; print(a[-1]); print(a[-3]); print(a[5:10]); print(a[-10:2])`
**WHEN** executed
**THEN** `30`, `10`, `[]`, `[10, 20]` — matches CPython slice clamping

### S5: List — step slicing (R3)

**GIVEN** `a = list(range(10)); print(a[::3]); print(a[1::2]); print(a[::-2])`
**WHEN** executed
**THEN** `[0, 3, 6, 9]`, `[1, 3, 5, 7, 9]`, `[9, 7, 5, 3, 1]`

### S6: List — concat, repeat, comprehension (R4)

**GIVEN** `print([1, 2] + [3, 4]); print([0] * 3); print([x**2 for x in range(5)])`
**WHEN** executed
**THEN** `[1, 2, 3, 4]`, `[0, 0, 0]`, `[0, 1, 4, 9, 16]`

### S7: List — empty list edge cases (R1, R2, R3)

**GIVEN** `a = []; print(len(a)); print(a[::-1]); print(list()); print([x for x in []])`
**WHEN** executed
**THEN** `0`, `[]`, `[]`, `[]`

### S8: Dict — access, mutation, KeyError (R5)

**GIVEN** `d = {'a': 1, 'b': 2}; print(d['a']); print(d.get('z', 99)); d['c'] = 3; del d['a']; print(len(d))`
**WHEN** executed
**THEN** `1`, `99`, `2` — `del` removes key, `get` returns default

### S9: Dict — pop, popitem, setdefault (R5)

**GIVEN** `d = {'x': 10, 'y': 20}; print(d.pop('x')); print(d.pop('z', -1)); d.setdefault('y', 99); d.setdefault('w', 30); print(d)`
**WHEN** executed
**THEN** `10`, `-1`, `{'y': 20, 'w': 30}` — `pop` with default, `setdefault` no-overwrite

### S10: Dict — merge operator PEP 584 (R6)

**GIVEN** `a = {'x': 1}; b = {'x': 2, 'y': 3}; print(a | b); a |= {'z': 4}; print(a)`
**WHEN** executed
**THEN** `{'x': 2, 'y': 3}`, `{'x': 1, 'z': 4}` — right side wins in merge

### S11: Dict — insertion order, keys/values/items (R6)

**GIVEN** `d = {}; d['c'] = 3; d['a'] = 1; d['b'] = 2; print(list(d.keys())); print(sorted(d.values()))`
**WHEN** executed
**THEN** `['c', 'a', 'b']`, `[1, 2, 3]` — insertion order preserved

### S12: Set — add, remove, discard, KeyError (R7)

**GIVEN** `s = {1, 2, 3}; s.add(4); s.discard(2); s.remove(3); print(sorted(s))`
**WHEN** executed
**THEN** `[1, 4]` — `remove` raises `KeyError` if absent, `discard` silently ignores

### S13: Set — algebra operators and methods (R8)

**GIVEN** `a = {1, 2, 3}; b = {2, 3, 4}; print(sorted(a | b)); print(sorted(a & b)); print(sorted(a - b)); print(sorted(a ^ b))`
**WHEN** executed
**THEN** `[1, 2, 3, 4]`, `[2, 3]`, `[1]`, `[1, 4]`

### S14: Set — subset, superset, frozenset (R8)

**GIVEN** `print({1, 2}.issubset({1, 2, 3})); print({1, 2, 3}.issuperset({1, 2})); fs = frozenset([3, 1, 2]); print(sorted(fs)); print(1 in fs)`
**WHEN** executed
**THEN** `True`, `True`, `[1, 2, 3]`, `True`

### S15: Tuple — indexing, comparison, membership (R9)

**GIVEN** `t = (1, 2, 3); print(t[0]); print(t[-1]); print((1, 2) < (1, 3)); print(2 in t); print((1,)); print(())`
**WHEN** executed
**THEN** `1`, `3`, `True`, `True`, `(1,)`, `()`

### S16: Tuple — count, index, concat, repeat, hash (R9)

**GIVEN** `t = (1, 2, 2, 3); print(t.count(2)); print(t.index(2)); print((1,) + (2, 3)); print((0,) * 3); print(hash((1, 2)) == hash((1, 2)))`
**WHEN** executed
**THEN** `2`, `1`, `(1, 2, 3)`, `(0, 0, 0)`, `True`

### S17: String — case methods (R10)

**GIVEN** `s = 'hello world'; print(s.upper()); print(s.lower()); print(s.capitalize()); print(s.title()); print(s.swapcase())`
**WHEN** executed
**THEN** `HELLO WORLD`, `hello world`, `Hello world`, `Hello World`, `HELLO WORLD`

### S18: String — strip, split, join, replace, find (R10)

**GIVEN** `print('  hi  '.strip()); print('a,b,c'.split(',')); print(','.join(['x','y'])); print('hello'.replace('l','r')); print('hello'.find('ll')); print('hello'.find('xyz'))`
**WHEN** executed
**THEN** `hi`, `['a', 'b', 'c']`, `x,y`, `herro`, `2`, `-1`

### S19: String — predicates and padding (R10)

**GIVEN** `print('123'.isdigit()); print('abc'.isalpha()); print(' '.isspace()); print('hi'.center(10, '-')); print('hi'.ljust(5)); print('42'.zfill(5))`
**WHEN** executed
**THEN** `True`, `True`, `True`, `----hi----`, `hi   `, `00042`

### S20: String — partition, removeprefix, removesuffix, splitlines (R10)

**GIVEN** `print('hello-world'.partition('-')); print('hello-world'.rpartition('-')); print('TestCase'.removeprefix('Test')); print('file.txt'.removesuffix('.txt')); print('a\nb\nc'.splitlines())`
**WHEN** executed
**THEN** `('hello', '-', 'world')`, `('hello', '-', 'world')`, `Case`, `file`, `['a', 'b', 'c']`

### S21: String — indexing and slicing (R11)

**GIVEN** `s = 'abcdef'; print(s[0]); print(s[-1]); print(s[1:4]); print(s[::-1]); print(s[::2])`
**WHEN** executed
**THEN** `a`, `f`, `bcd`, `fedcba`, `ace`

### S22: Bytes — construction, decode, hex (R12)

**GIVEN** `b = bytes([72, 101, 108, 108, 111]); print(b); print(b.decode('utf-8')); print(b.hex()); b2 = bytes.fromhex('48656c6c6f'); print(b2)`
**WHEN** executed
**THEN** `b'Hello'`, `Hello`, `48656c6c6f`, `b'Hello'`

### S23: Bytes — find, startswith, split, strip, replace (R12)

**GIVEN** `b = b'Hello, World!'; print(b.find(b'World')); print(b.startswith(b'Hello')); print(b'a,b,c'.split(b',')); print(b'  hi  '.strip()); print(b'hello'.replace(b'l', b'r'))`
**WHEN** executed
**THEN** `7`, `True`, `[b'a', b'b', b'c']`, `b'hi'`, `b'herro'`

### S24: Bytearray — mutable operations (R12)

**GIVEN** `ba = bytearray(b'abc'); ba[0] = 65; ba.append(100); ba.extend(b'ef'); print(ba); v = ba.pop(); print(v); ba.reverse(); print(ba)`
**WHEN** executed
**THEN** mutable ops produce identical results to CPython 3.12
