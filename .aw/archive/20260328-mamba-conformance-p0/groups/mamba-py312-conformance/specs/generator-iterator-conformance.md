# Generator Iterator Conformance

## Overview

Conformance test suite ensuring Mamba generator and iterator protocol matches CPython 3.12 behavior. Covers #756.

| Protocol Area | Impl Module | Operations Covered | Existing Fixtures |
|---------------|-------------|-------------------|-------------------|
| Basic yield | `runtime/generator.rs` | yield, yield without value, generator exhaustion, for-loop iteration, list() conversion, manual next() | 1 (basic_yield) |
| send/throw/close | `runtime/generator.rs` | send(value), throw(exc_type, value), close(), GeneratorExit handling | 1 (send_throw) |
| StopIteration.value | `runtime/generator.rs` | return from generator, StopIteration.value propagation, bare return, no return | 1 (stopiteration) |
| yield from | `runtime/generator.rs` | delegation to sub-generator, delegation to iterable, return value capture, nested delegation, yield from string/list | 1 (yield_from) |
| Iterator protocol | `runtime/iter.rs` | __iter__, __next__, StopIteration, iter() on builtins, next() with default, user-defined iterators | 1 (protocol) |
| Generator state | `runtime/generator.rs` | gi_frame, created/suspended/closed states | partial (language/generators) |
| Async generators | `runtime/generator.rs` | async yield, async for | 0 (xfail — not implemented) |
| Iteration builtins | `runtime/iter.rs` | enumerate, zip, map, filter, reversed, iter(callable, sentinel) | 1 (builtins/iteration) |

**Test approach**: Each `.py` fixture in `tests/fixtures/conformance/generators/` and `tests/fixtures/conformance/iterators/` is executed through the full Mamba JIT pipeline (parse → typecheck → HIR → MIR → Cranelift JIT → capture stdout). Output is compared against `.expected` golden files generated by CPython 3.12. Any diff = conformance bug.

**Scope**: Existing fixtures cover basic generator and iterator operations. This spec adds comprehensive edge-case fixtures for generator expressions, chained yield-from, generator-based context managers, iterator composition (zip, enumerate, map, filter with generators), and error propagation edge cases to reach full CPython 3.12 parity.
## Requirements

| ID | Type | Title | Priority | Acceptance Criteria |
|----|------|-------|----------|---------------------|
| R1 | generator | Generator expressions match CPython | P0 | `(x for x in iterable)`, `(x for x in it if pred)`, nested genexpr, genexpr as function arg (e.g. `sum(x**2 for x in range(5))`) produce identical stdout to CPython 3.12 |
| R2 | generator | Generator send edge cases match CPython | P0 | `send(None)` on unstarted generator, `send(value)` before first yield raises `TypeError`, send to exhausted generator raises `StopIteration` — identical behavior |
| R3 | generator | Generator throw edge cases match CPython | P0 | `throw(TypeError)` with no active except, `throw` into finally block, `throw` on exhausted generator — identical exception propagation |
| R4 | generator | Generator close edge cases match CPython | P0 | `close()` on unstarted generator, `close()` on exhausted generator (no-op), `close()` when generator ignores GeneratorExit raises `RuntimeError` — identical behavior |
| R5 | generator | yield from with send/throw passthrough match CPython | P0 | `send(value)` through yield-from chain reaches inner generator, `throw(exc)` through yield-from reaches inner generator, inner generator return value captured — identical behavior |
| R6 | generator | Generator state attributes match CPython | P0 | `gi_frame`, `gi_running`, `gi_code`, `gi_yieldfrom` attributes present and match CPython 3.12 state transitions (created → suspended → running → closed) |
| R7 | iterator | Custom iterator classes match CPython | P0 | Classes implementing `__iter__`/`__next__` work with `for`, `list()`, `next()`, `in`, unpacking — identical results; `StopIteration` from `__next__` terminates iteration |
| R8 | iterator | Iterator composition with generators match CPython | P0 | `enumerate(gen())`, `zip(gen1(), gen2())`, `map(fn, gen())`, `filter(fn, gen())`, `reversed(list(gen()))`, chained compositions — identical results |
| R9 | iterator | iter() two-argument form match CPython | P0 | `iter(callable, sentinel)` stops when callable returns sentinel, works with generators and closures — identical behavior |
| R10 | generator | Generator-based context manager match CPython | P0 | `contextlib.contextmanager`-style pattern (yield in try/finally), exception forwarding into generator, cleanup on exception — identical behavior |
| R11 | generator | Generator memory/lifecycle match CPython | P0 | Generator garbage collection triggers `close()`, `del` on generator triggers `close()`, generator with pending finally block executes finally on close — identical behavior |
| R12 | iterator | Iterable unpacking with generators match CPython | P0 | `a, b, c = gen()`, `a, *rest = gen()`, `first, *mid, last = gen()` — identical results; `ValueError` on size mismatch |

### Constraints

- All existing conformance fixtures in `tests/fixtures/conformance/generators/` and `tests/fixtures/conformance/iterators/` must continue to pass
- Test harness: `.py` fixture + `.expected` golden file, run via `conformance_tests.rs`
- Golden files generated by running fixtures under CPython 3.12
- Each new fixture must cover edge cases not in existing fixtures
- No changes to runtime implementation in this spec — test-only; implementation bugs found are tracked as separate issues
- Async generators are out of scope (xfail — tracked separately in #800)
## Scenarios

### S1: Generator expression — basic and filtered (R1)

**GIVEN** `print(list(x**2 for x in range(5))); print(list(x for x in range(10) if x % 2 == 0)); print(sum(x for x in range(4)))`
**WHEN** executed through Mamba JIT
**THEN** `[0, 1, 4, 9, 16]`, `[0, 2, 4, 6, 8]`, `6`

### S2: Generator expression — nested (R1)

**GIVEN** `print(list((x, y) for x in range(3) for y in range(2)))`
**WHEN** executed
**THEN** `[(0, 0), (0, 1), (1, 0), (1, 1), (2, 0), (2, 1)]`

### S3: Generator expression — as function argument (R1)

**GIVEN** `print(max(x**2 for x in range(-3, 4))); print(min(abs(x) for x in [-5, 3, -1, 4]))`
**WHEN** executed
**THEN** `9`, `1`

### S4: send(None) to prime and send before first yield (R2)

**GIVEN** generator `def g(): val = yield 1; yield val * 2`; call `g2 = g(); print(g2.send(None))` to prime; separately test `g3 = g(); g3.send(42)` expecting TypeError
**WHEN** executed
**THEN** `1` for send(None) prime; `TypeError: can't send non-None value to a just-started generator` for send(42) on unstarted

### S5: send to exhausted generator (R2)

**GIVEN** `def g(): yield 1`; exhaust it, then `g.send(1)` expecting StopIteration
**WHEN** executed
**THEN** `StopIteration` raised

### S6: throw into generator without matching except (R3)

**GIVEN** `def g(): yield 1; yield 2`; advance once, then `g.throw(TypeError, 'bad')` 
**WHEN** executed
**THEN** `TypeError: bad` propagated to caller (generator terminates)

### S7: throw into finally block (R3)

**GIVEN** `def g(): try: yield 1 finally: print('cleanup')`; advance once, then throw ValueError
**WHEN** executed
**THEN** `cleanup` printed, then ValueError propagated

### S8: throw on exhausted generator (R3)

**GIVEN** generator exhausted, then `g.throw(RuntimeError)`
**WHEN** executed
**THEN** `RuntimeError` raised immediately (no GeneratorExit)

### S9: close() on unstarted and exhausted generator (R4)

**GIVEN** `def g(): yield 1; yield 2`; `g1 = g(); g1.close()` (unstarted); `g2 = g(); next(g2); next(g2); g2.close()` (exhausted after StopIteration caught)
**WHEN** executed
**THEN** Both close() calls succeed silently (no-op)

### S10: close() when generator ignores GeneratorExit (R4)

**GIVEN** `def g(): try: yield 1 except GeneratorExit: yield 2`; advance once then close()
**WHEN** executed
**THEN** `RuntimeError: generator ignored GeneratorExit`

### S11: yield from — send passthrough (R5)

**GIVEN** `def inner(): val = yield 'ready'; yield val * 10`; `def outer(): result = yield from inner()`; prime outer, send(5)
**WHEN** executed
**THEN** `ready` from first next, `50` from send(5) — send passes through yield-from to inner

### S12: yield from — throw passthrough (R5)

**GIVEN** `def inner(): try: yield 1 except ValueError as e: yield str(e)`; `def outer(): yield from inner()`; advance, throw ValueError
**WHEN** executed
**THEN** throw passes through to inner, inner catches and yields error message

### S13: yield from — return value capture (R5)

**GIVEN** `def inner(): yield 1; return 42`; `def outer(): result = yield from inner(); print('got:', result); yield result`
**WHEN** executed
**THEN** `got: 42` printed, outer yields `42`

### S14: Generator state attributes (R6)

**GIVEN** `def g(): yield 1; yield 2`; check `gi_frame` at created, suspended, closed states
**WHEN** executed
**THEN** `gi_frame is not None` when created/suspended, `gi_frame is None` when closed — matches CPython

### S15: Custom iterator class — for loop and list() (R7)

**GIVEN** `class Fib: __init__(self, n), __iter__(self), __next__(self)` implementing Fibonacci up to n terms
**WHEN** `list(Fib(6))` and `for x in Fib(4): print(x)`
**THEN** identical Fibonacci output to CPython 3.12

### S16: Custom iterator — unpacking and membership (R7)

**GIVEN** custom iterator yielding [10, 20, 30]; `a, b, c = MyIter()`; `print(20 in MyIter())`
**WHEN** executed
**THEN** `a=10, b=20, c=30`, `True`

### S17: enumerate and zip with generators (R8)

**GIVEN** `def gen(): yield 'a'; yield 'b'; yield 'c'`; `print(list(enumerate(gen())))`; `print(list(zip(gen(), range(3))))`
**WHEN** executed
**THEN** `[(0, 'a'), (1, 'b'), (2, 'c')]`, `[('a', 0), ('b', 1), ('c', 2)]`

### S18: map and filter with generators (R8)

**GIVEN** `def nums(): yield 1; yield 2; yield 3; yield 4`; `print(list(map(lambda x: x*2, nums())))`; `print(list(filter(lambda x: x%2==0, nums())))`
**WHEN** executed
**THEN** `[2, 4, 6, 8]`, `[2, 4]`

### S19: Chained iterator composition (R8)

**GIVEN** `print(list(enumerate(filter(lambda x: x>0, map(lambda x: x-2, [1,2,3,4,5])))))`
**WHEN** executed
**THEN** `[(0, 1), (1, 2), (2, 3)]`

### S20: iter(callable, sentinel) (R9)

**GIVEN** `vals = iter([3, 2, 1, 0]); print(list(iter(lambda: next(vals), 0)))`
**WHEN** executed
**THEN** `[3, 2, 1]` — stops before sentinel

### S21: Generator-based context manager pattern (R10)

**GIVEN** generator with try/yield/finally for resource management; use pattern: acquire → yield → release
**WHEN** executed with and without exception in body
**THEN** cleanup runs in both cases, matching CPython

### S22: Generator lifecycle — close on del (R11)

**GIVEN** `def g(): try: yield 1 finally: print('finalized')`; `g1 = g(); next(g1); del g1`
**WHEN** executed
**THEN** `finalized` printed when generator is deleted

### S23: Generator with pending finally on close (R11)

**GIVEN** `def g(): try: yield 1; yield 2 finally: print('finally ran')`; advance once, then close()
**WHEN** executed
**THEN** `finally ran` printed — finally block executes on close()

### S24: Iterable unpacking with generators (R12)

**GIVEN** `def g(): yield 1; yield 2; yield 3`; `a, b, c = g(); print(a, b, c)`; `first, *rest = g(); print(first, rest)`; `a, *mid, last = g(); print(a, mid, last)`
**WHEN** executed
**THEN** `1 2 3`, `1 [2, 3]`, `1 [2] 3`

### S25: Unpacking size mismatch with generator (R12)

**GIVEN** `def g(): yield 1; yield 2`; `a, b, c = g()` expecting ValueError
**WHEN** executed
**THEN** `ValueError: not enough values to unpack`

### Strategy

|-------|-----------|----------|
| Fixture conformance | `.py` fixture + `.expected` golden file, run via `conformance_tests.rs` harness | `tests/fixtures/conformance/generators/`, `tests/fixtures/conformance/iterators/` |
