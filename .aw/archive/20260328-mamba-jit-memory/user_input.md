Replace cranelift-jit dependency with custom JIT memory manager using mmap(MAP_JIT) on macOS aarch64. cranelift-jit uses alloc::alloc + region::protect(mprotect) which causes EXC_BAD_ACCESS code=257 (PAC failure) after ~100 cross-thread JIT executions on Apple Silicon. New MambaJitModule wraps cranelift-codegen directly: compile functions via cranelift Context, allocate executable pages with mmap(MAP_JIT) + pthread_jit_write_protect_np, handle symbol resolution and relocations manually. Drop cranelift-jit from Cargo.toml. Keep cranelift-codegen + cranelift-frontend + cranelift-module. #1114
