Replace per-generator thread::spawn with a reusable worker thread pool in generator.rs. After ~130 sequential generator thread spawns, macOS aarch64 hits EXC_BAD_ACCESS code=257 (PAC failure) when calling JIT code. This blocks enabling any new conformance tests (stuck at 99 active / 50 xfail). Root cause: cumulative pthread lifecycle operations corrupt process state on Apple Silicon. Fix: use a long-lived worker thread that receives generator tasks via channel, avoiding repeated thread creation/destruction.
