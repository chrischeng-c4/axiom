# Requirements

Implement the full AOT production build pipeline (#765) across 7 milestones: (1) Tree shaking — ESM static import/export graph analysis, package.json sideEffects field support, DCE for unreachable code paths, scope-level variable reference tracking, barrel file re-export flattening. (2) Code splitting — dynamic import() creates separate async chunks, shared-chunk extraction for modules used by 2+ importers, multi-entry-point support, [name].[hash].js output naming, modulepreload hint generation, manual chunk config. (3) Minification — identifier mangling, whitespace removal, constant folding, CSS minification, HTML minification, console.log/debugger removal in production, scope-aware name mangling. (4) Source maps — VLQ-encoded per-transformation mappings, composition chain (TS→JS→minified→bundled), inline (data: URL) and external (.map file) modes, sourcesContent embedding. (5) CSS pipeline — @import resolution, CSS Modules (.module.css) with scoped class names + JS object export, PostCSS plugin pipeline, Tailwind JIT, asset URL rewriting with hashed output. (6) Asset pipeline — image optimization, font copy+hash with @font-face generation, SVG inline-as-component or copy, JSON tree-shaking, WASM bundling, public/ directory with hashing. (7) Build config — process.env.NODE_ENV replacement, define (compile-time constants), ES target level, module format output (ESM/CJS/UMD/IIFE), library mode preserving exports, externals for SSR/library use, tsconfig.json path alias resolution. Output: dist/ with index.html, assets/main.[hash].js, assets/main.[hash].css, .map files. Validate (#797) by building TodoMVC React and Realworld React+Redux with jet, running Playwright smoke tests comparing against Vite/webpack output, and documenting any unsupported patterns as known limitations. Acceptance: ≥2 real-world projects build and run with no JS runtime errors.
