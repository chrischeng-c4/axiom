# Python Code Generator Specification

## Overview

Extends the existing `cclab prism gen-stub` to generate complete Python wrapper code,
not just `.pyi` stubs. The goal is to make Python code fully auto-generated from Rust.

## Generation Flow

```
┌─────────────────────────────────────────────────────────────┐
│                    cclab prism gen-python                    │
├─────────────────────────────────────────────────────────────┤
│  Input: Rust source with PyO3 annotations                    │
│  Output: Python package with:                                │
│    - __init__.py (re-exports)                               │
│    - _types.py (type definitions)                           │
│    - __init__.pyi (type stubs)                              │
└─────────────────────────────────────────────────────────────┘
```

## CLI Interface

```bash
cclab prism gen-python <RUST_CRATE_PATH> [OPTIONS]

Arguments:
  <RUST_CRATE_PATH>  Path to Rust crate with PyO3 bindings

Options:
  -o, --output <DIR>       Output directory for Python package
      --module-name <NAME> Python module name (default: crate name)
      --include-stubs      Also generate .pyi stub files (default: true)
      --thin-wrapper       Generate minimal Python wrapper
      --dry-run            Print generated code without writing
```

## Output Structure

For `cclab prism gen-python crates/cclab-titan -o python/cclab/titan`:

```
python/cclab/titan/
├── __init__.py       # Re-exports from Rust
├── __init__.pyi      # Type stubs
└── _types.py         # Python type aliases (if needed)
```

## Implementation in cclab-prism

### Enhance Existing PyO3StubGenerator

```rust
// crates/cclab-prism/src/gen/python/mod.rs

pub struct PythonCodeGenerator {
    stub_gen: PyO3StubGenerator,
    config: PythonGenConfig,
}

#[derive(Debug, Clone)]
pub struct PythonGenConfig {
    pub module_name: String,
    pub include_stubs: bool,
    pub thin_wrapper: bool,
    pub output_dir: PathBuf,
}

impl PythonCodeGenerator {
    pub fn generate(&self, rust_source: &Path) -> Result<GeneratedPython> {
        // 1. Parse Rust source with tree-sitter
        let entities = self.stub_gen.extract_pyo3_entities(source)?;

        // 2. Generate __init__.py
        let init_py = self.generate_init_py(&entities)?;

        // 3. Generate __init__.pyi (using existing stub generator)
        let init_pyi = self.stub_gen.generate_stub(&entities)?;

        Ok(GeneratedPython {
            init_py,
            init_pyi,
        })
    }

    fn generate_init_py(&self, entities: &PyO3Entities) -> Result<String> {
        let mut output = String::new();

        // Header
        output.push_str(&format!(
            r#""""
{} - Auto-generated from Rust.

DO NOT EDIT - Generated by `cclab prism gen-python`.
"""
from __future__ import annotations

"#,
            self.config.module_name
        ));

        // Import from Rust module
        output.push_str(&format!(
            "from cclab._nucleus.{} import (\n",
            self.config.module_name
        ));

        // List all exports
        for class in &entities.classes {
            output.push_str(&format!("    {},\n", class.name));
        }
        for func in &entities.functions {
            output.push_str(&format!("    {},\n", func.name));
        }
        output.push_str(")\n\n");

        // __all__
        output.push_str("__all__ = [\n");
        for class in &entities.classes {
            output.push_str(&format!("    \"{}\",\n", class.name));
        }
        for func in &entities.functions {
            output.push_str(&format!("    \"{}\",\n", func.name));
        }
        output.push_str("]\n");

        Ok(output)
    }
}
```

## Validation

Generated Python code is validated by:

1. `python -m py_compile <file>` - Syntax check
2. `mypy --strict <file>` - Type checking
3. `import cclab.titan` - Import test
