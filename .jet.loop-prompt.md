**Goal**: find drift AND fix it in the same tick. Spec-align is NOT read-only.

**Mode selection** (PM decides before dispatch):

- **Mode A — fix-known**: the target file has recorded drifts in `.jet.state` `drifts_found[]` with `status: open*`.
  Pick the highest-severity open drift for that file.
  - **Brief**: "Read spec `<path>`. Fix this drift: `<claim>` — expected `<expected>`, actual `<actual>`. Edit the spec file to match current source code reality. Rules: (1) Strip or correct the inaccurate claim — do NOT invent new numbers, grep/wc to get current values. (2) If drift is LOC-table: remove the entire LOC table (systemic — tables go stale). (3) If drift is META self-ref (main_spec_ref points to itself): update frontmatter `main_spec_ref` to point to the primary source file (e.g., `crates/cclab-jet/src/bundler/mod.rs`). (4) If drift is plan-not-executed: change requirement status from `implemented` to `planned`, or add a note `<!-- status: planned, not yet wired -->`. (5) If drift is function-name: update the spec identifier to match the actual function name. (6) Do NOT alter source code — only edit spec files. Under 200 words summary of changes."
  - **QA bar**: spec file was edited, the specific drift claim is corrected, no unrelated content was removed, file still valid markdown. Grep to verify the fix matches reality.
  - **On pass**: update drift entry in `.jet.state` to `status: fixed-tick-<N>`, do NOT delete it (audit trail).

- **Mode B — discover**: no open drifts for the target file (all fixed or file is clean).
  - **Brief**: same as before — "Read `<path>`. Spot-check 2-3 concrete claims against `crates/cclab-jet/src/`. Return drift list as `{claim, expected, actual, severity}` or `no drift`. Under 200 words."

  - **On drift found**: add to `drifts_found[]` in state. Next tick for this file will be Mode A.

**Target rotation**: next file alphabetically after `last_file_reviewed`. But if ANY file has open high/critical drifts, prioritize that file first (fix before discover).
